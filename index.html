termux ajuda

Termux é um emulador de terminal Android e aplicativo de ambiente Linux que funciona diretamente sem enraizamento ou configuração necessária. Um sistema básico mínimo é instalado automaticamente, pacotes adicionais estão disponíveis usando o gerenciador de pacotes

Como funciona?

O emulador de terminal é basicamente um aplicativo que inicia o programa de linha de comando usando execve de chamada de sistema e redirecionando fluxos de entrada, saída e erros padrão para a tela.

A maioria dos aplicativos de terminal disponíveis no sistema operacional Android funcionam com um conjunto muito limitado de utilitários que geralmente são fornecidos pelo sistema operacional ou por outras ferramentas de root, como Magisk. Decidimos ir mais longe e portar software comum geralmente disponível em sistemas GNU / Linux para o sistema operacional Android.

O Termux não é uma máquina virtual nem qualquer outro tipo de ambiente emulado ou simulado. Todos os pacotes fornecidos são compilados com o Android NDK e só têm patches de compatibilidade para funcionar no Android. O sistema operacional não fornece acesso total aos seus sistemas de arquivos, portanto o Termux não pode instalar arquivos de pacote em diretórios padrão como / bin, / etc, / usr ou / var. Em vez disso, todos os arquivos são instalados no diretório do aplicativo privado localizado em

/data/data/com.termux/files/usr

Chamamos esse diretório de "prefixo" e normalmente nos referimos a ele como "$ PREFIX", que também é uma variável de ambiente exportada no shell do Termux. Observe que este diretório não pode ser alterado ou movido para um cartão SD porque:

  • O sistema de arquivos deve ter suporte para permissões unix e arquivos especiais como links simbólicos ou sockets.
  • O caminho do prefixo é codificado em todos os binários.

Além do prefixo, os usuários podem armazenar arquivos no diretório inicial (ou "$ HOME") disponível em

/data/data/com.termux/files/home

O que posso fazer com o Termux?

Existem vários casos de uso comuns para o aplicativo Termux:

  • Processamento de dados com Python.

  • Programação em ambiente de desenvolvimento.

  • Baixar e gerenciar arquivos e páginas usando ferramentas estabelecidas pelo tempo.

  • Aprendendo o básico do ambiente de linha de comando do Linux.

  • Executando um cliente SSH.

  • Sincronizando e fazendo backup de seus arquivos

Obviamente, o uso não se limita aos tópicos listados acima. Existem mais de 1000 pacotes em nossos repositórios. Se esses pacotes não tiverem o que você procura, você pode compilar o seu próprio - temos uma variedade de ferramentas de construção, incluindo compiladores para linguagens como C, C++, Go, Rust. Intérpretes para linguagens comuns como NodeJS, Python, Ruby também estão disponíveis.

Observe que o Termux não é uma ferramenta de root e não lhe dará privilégios de root, a menos que você seja qualificado o suficiente para quebrar a segurança do sistema operacional Android.

É necessário root?

Normalmente o Termux não requer que o dispositivo seja enraizado. Na verdade, ele é direcionado principalmente para usuários não root.

Dicas

Aqui estão algumas dicas básicas sobre como usar o Termux e sobreviver:

  • Aprenda shell scripting!

  • Sempre mantenha seus pacotes atualizados! Execute o comando pkg upgrade regularmente ou pelo menos antes de instalar um novo pacote.

  • Faça backups, sempre! Sem backups, você provavelmente não conseguirá reverter se algo der errado. Observe que os desenvolvedores de software devem prestar atenção em fazer backup de debfiles de compiladores, interpretadores ou dependências usados porque o Termux não fornece versões de pacote mais antigas e é um lançamento contínuo.

  • Não execute coisas que você não conhece! Revise os scripts baixados da Internet. Sempre pense no que você está digitando no terminal.

  • Leia atentamente tudo o que foi impresso no terminal! Compreender as mensagens informativas ajuda a resolver problemas que podem ocorrer.

Não somos capazes de manter toda a documentação sobre comandos do Linux, scripts de shell e outras informações de uso geral, portanto, links para recursos externos são fornecidos.

Sugere fortemente evitar os tutoriais do YouTube, especialmente relacionados a hackers. Existem muitos clickbait direcionados a usuários não experientes. Se você decidir segui-los, certifique-se de compreender os comandos executados. Além disso, sempre verifique o conteúdo dos arquivos baixados. Se o conteúdo dos scripts baixados for ofuscado, deve haver alarme sobre conteúdo potencialmente inseguro.

pwd (Imprimir diretório de trabalho)

Tudo no Linux é um arquivo, conforme você se aprofunda no Linux, você entenderá isso, mas por enquanto apenas mantenha isso em mente. Cada arquivo é organizado em uma árvore de diretório hierárquica. O primeiro diretório no sistema de arquivos é apropriadamente chamado de diretório raiz. O diretório raiz tem muitas pastas e arquivos que você pode armazenar mais pastas e arquivos, etc. Aqui está um exemplo de como a árvore de diretórios se parece:

A localização desses arquivos e diretórios é conhecida como caminhos. Se você tivesse uma pasta chamada home com uma pasta dentro dela chamada pete e outra pasta nessa pasta chamada Movies, esse caminho seria assim: /home/pete/Movies, muito simples não?

digite no termux

pwd

cd (Alterar diretório)

l

Agora que você sabe onde está, vamos ver se podemos mover um pouco o sistema de arquivos. Lembre-se de que precisamos navegar nosso caminho usando caminhos. Existem duas maneiras diferentes de especificar um caminho, com caminhos absolutos e relativos.

  • Caminho absoluto: este é o caminho do diretório raiz. A raiz é o chefão. O diretório raiz normalmente é mostrado como uma barra. Cada vez que seu caminho começa com /, significa que você está começando no diretório raiz. Por exemplo, /home/pete/Desktop.

  • Caminho relativo: Este é o caminho de onde você está atualmente no sistema de arquivos. Se eu estivesse no local /home/pete/Documents e quisesse acessar um diretório dentro de Documents chamado de impostos, não preciso especificar o caminho completo da raiz como /home/pete/Documentos/impostos, posso apenas ir para /impostos em vez disso.

Agora que você sabe como funcionam os caminhos, só precisamos de algo que nos ajude a mudar para o diretório que desejamos. Felizmente, temos cd ou “mudar de diretório” para fazer isso.

cd /home/pete/Imagens

Agora mudei a localização do meu diretório para /home/pete/Pictures.
Agora, a partir deste diretório, tenho uma pasta chamada Hawaii, posso navegar até essa pasta com:

cd Hawaii

Percebeu como acabei de usar o nome da pasta? É porque eu já estava em /home/pete/Pictures.
Pode ser muito cansativo navegar com caminhos absolutos e relativos o tempo todo, felizmente existem alguns atalhos para ajudá-lo.

  • (diretório atual). Este é o diretório em que você está.

  • (Diretório Parental). Leva você ao diretório acima do atual.

  • (diretório inicial). Este diretório é padronizado como seu “diretório inicial”. Como /home/pete.

  • (diretório anterior). Isso o levará para o diretório anterior em que estava.

cd .
cd ..
cd ~
cd -

ls (lista de diretórios)

Agora que sabemos como se mover no sistema, como descobrimos o que está disponível para nós? No momento, é como se estivéssemos nos movendo no escuro. Bem, podemos usar o maravilhoso comando ls para listar o conteúdo do diretório. O comando ls listará diretórios e arquivos no diretório atual por padrão, no entanto, você pode especificar de qual caminho deseja listar os diretórios.

ls
ls /home/pete

ls é uma ferramenta bastante útil, mas também mostra informações detalhadas sobre os arquivos e diretórios que você está procurando.
Observe também que nem todos os arquivos em um diretório estarão visíveis. Nomes de arquivos que começam com . estão ocultos, você pode visualizá-los com o comando ls e passar o sinalizador -a para ele.

ls -a

Há também um sinalizador ls mais útil, -l que mostra uma lista detalhada de arquivos em um formato longo. Isso mostrará informações detalhadas, começando da esquerda: permissões do arquivo, número de links, nome do proprietário, grupo de proprietários, tamanho do arquivo, data e hora da última modificação e nome do arquivo /diretório.

ls -l

total 80
drwxr-x --- 7 pete penguingroup 4096 20 de novembro 16:37 Desktop
drwxr-x --- 2 pete penguingroup 4096 19 de outubro 10:46 Documentos
drwxr-x --- 4 pete penguingroup 4096 20 de novembro 09:30 Downloads
drwxr-x --- 2 pete penguingroup 4096 7 de outubro 13:13 Música
drwxr-x --- 2 pete penguingroup 4096 21 de setembro 14:02 Imagens
drwxr-x --- 2 pete penguingroup 4096 Jul 27 12:41 Público
drwxr-x --- 2 pete penguingroup 4096 Jul 27 12:41 Modelos
drwxr-x --- 2 pete penguingroup 4096 Jul 27 12:41 Vídeos

Os comandos têm coisas chamadas sinalizadores (ou argumentos ou opções, como você quiser chamá-los) para adicionar mais funcionalidade. Veja como adicionamos -a e -l, bem, você pode adicioná-los junto com -la. A ordem dos sinalizadores determina em qual ordem eles entram, na maioria das vezes isso realmente não importa, então você também pode fazer ls -al e ainda funcionaria.

ls -la

  • ls -R: lista recursivamente o conteúdo do diretório

  • ls -r: ordem inversa durante a classificação

  • ls -t: classificar por hora de modificação, mais recente primeiro

touch

Vamos aprender como fazer alguns arquivos. Uma maneira muito simples é usar o comando de toque. touch
permite que você crie novos arquivos vazios.

touch senhas.txt

O toque também é usado para alterar os carimbos de data hora em arquivos e diretórios existentes. Experimente, faça um ls -l em um arquivo e anote o carimbo de data hora, faça digite touch nesse arquivo e ele atualizará a data hora da modificação.
Existem muitas outras maneiras de criar arquivos que envolvem outras coisas, como redirecionamento e editores de texto, mas vamos chegar a isso no curso Manipulação de Texto.

cat

Um comando simples de usar é o comando cat, abreviação de concatenar, ele não apenas exibe o conteúdo do arquivo, mas pode combinar vários arquivos e mostrar a saída deles.

cat script1.sh script2.py

Não é ótimo para visualizar arquivos grandes e serve apenas para conteúdo curto. Existem muitas outras ferramentas que usamos para visualizar arquivos de texto maiores que discutiremos na próxima lição.

less

Se você estiver visualizando arquivos de texto maiores do que uma saída simples, less e o comando ideal. (Na verdade, existe um comando chamado more que faz algo semelhante, então isso é irônico.) O texto é exibido de forma paginada, para que você possa navegar por um arquivo de texto página por página. Vá em frente e observe o conteúdo de um arquivo com less. Assim que estiver no comando less, você pode usar outros comandos de teclado para navegar no arquivo.

less /home/pete/Documents/text1

Use o seguinte comando para navegar por less:

  • q sair do less e voltar ao seu shell.
  • g vai para o início do arquivo de texto.
  • G vai para o final do arquivo de texto.
  • / search Você pode pesquisar um texto específico dentro do documento de texto.
  • h Se precisar de ajuda para usar less enquanto usa less, use a ajuda.

history

Em seu shell, há um histórico dos comandos que você inseriu anteriormente, você pode realmente ver esses comandos. Isso é bastante útil quando você deseja localizar e executar um comando usado anteriormente sem realmente digitá-lo novamente.

history

Deseja executar o mesmo comando que você fez antes, basta clicar na seta para cima. Quer executar o comando anterior sem digitá-lo novamente? Usar !!. Se você digitou cat arquivo1 e deseja executá-lo novamente, você pode simplesmente ir !! e ele executará o último comando executado. Outro atalho de histórico é o ctrl-R, este é o comando de pesquisa reversa, se você clicar em ctrl-R e começar a digitar as partes do comando que deseja, ele mostrará as correspondências e você poderá navegar por elas pressionando a tecla ctrl-R novamente. Depois de encontrar o comando que deseja usar novamente, basta pressionar a tecla Enter. Nosso terminal está ficando um pouco confuso, não? Vamos fazer uma pequena limpeza, use o comando clear para limpar sua tela.

clear

Isso parece melhor, não é? Enquanto estamos falando sobre coisas úteis, um dos recursos mais úteis em qualquer ambiente de linha de comando é o preenchimento de tabulação. Se você começar a digitar o início de um comando, arquivo, diretório, etc e pressionar a tecla Tab, ele será preenchido automaticamente com base no que encontrar no diretório que você está pesquisando, desde que você não tenha nenhum outro arquivo que comece com aqueles cartas. Por exemplo, se você estava tentando executar o comando chrome, você pode digitar chr e pressionar Tab e ele irá preencher automaticamente o chrome.

cp (cópia)

Vamos começar a fazer algumas cópias desses arquivos. Muito parecido com copiar e colar arquivos em outros sistemas operacionais, o shell nos oferece uma maneira ainda mais simples de fazer isso.

cp /sdcard/Music/mc_mark.m4a

mc_mark.m4a é o arquivo que você deseja copiar e /data/data/com.termux/files/home é para onde você está copiando o arquivo. Você pode copiar vários arquivos e diretórios, bem como usar curingas. Um curinga é um caractere que pode ser substituído por uma seleção baseada em padrão, dando a você mais flexibilidade nas pesquisas. Você pode usar curingas em cada comando para obter mais flexibilidade.

  • * o curinga de curingas, é usado para representar todos os caracteres únicos ou qualquer string.

  • ? usado para representar um personagem

  • [] usado para representar qualquer caractere entre colchetes

cp * .jpg /sdcard/fotos

Isso copiará todos os arquivos com a extensão .jpg em seu diretório atual para o diretório fotos. Um comando útil é usar o sinalizador -r, isso copiará recursivamente os arquivos e diretórios dentro de um diretório.
Tente fazer um cp em um diretório que contém alguns arquivos para o diretório de documentos. Não funcionou, não é? Bem, isso é porque você precisará copiar os arquivos e diretórios internos também com o comando -r.

cp -r Abóbora / / home / pete / Documentos

Uma coisa a ser observada: se você copiar um arquivo para um diretório com o mesmo nome de arquivo, o arquivo será sobrescrito com o que você estiver copiando. Isso não é bom se você tiver um arquivo que não deseja que seja sobrescrito acidentalmente. Você pode usar o sinalizador -i (interativo) para avisar antes de sobrescrever um arquivo.

Imagens $ cp -i mycoolfile /home/pete/

mv (mover)

Usado para mover arquivos e também renomeá-los. Muito semelhante ao comando cp em termos de sinalizadores e funcionalidade. Você pode renomear arquivos assim:

mv musica.mp3 mc_mark.mp3

Ou você pode realmente mover um arquivo para um diretório diferente:

mv virus.apk /sdcard

E mova mais de um arquivo:

mv arquivo_1 arquivo_2 /algum_diretório

Como o cp, se você mv um arquivo ou diretório, ele sobrescreverá qualquer coisa no mesmo diretório. Portanto, você pode usar o sinalizador -i para avisar antes de sobrescrever qualquer coisa.

mv -i diretório1 diretório2

Digamos que você queira mover um arquivo para sobrescrever o anterior. Você também pode fazer um backup desse arquivo e ele apenas renomeará a versão antiga com um ~.

mv -b diretório1 diretório2

mkdir (Criar diretório)

Precisaremos de alguns diretórios para armazenar todos esses arquivos em que estamos trabalhando. O comando mkdir (Make Directory) é útil para isso, ele criará um diretório se ainda não existir. Você pode até criar vários diretórios ao mesmo tempo.

mkdir musicas videos

Você também pode criar subdiretórios ao mesmo tempo com o -p (sinalizador pai).

mkdir -p books/html/favoritos

rm (Remover)

Agora acho que temos muitos arquivos, vamos remover alguns arquivos. Para remover arquivos, você pode usar o comando rm. O comando rm (remover) é usado para excluir arquivos e diretórios.

rm arquivo1

Tome cuidado ao usar o rm, não há nenhuma lixeira mágica que você possa pescar nos arquivos removidos. Uma vez que eles se foram, eles se foram para sempre, então tome cuidado.
Felizmente, existem algumas medidas de segurança postas em prática, então o voce não pode simplesmente remover um monte de arquivos importantes. Arquivos protegidos contra gravação solicitarão confirmação antes de excluí-los. Se um diretório estiver protegido contra gravação, também não será facilmente removido.
Agora, se você não se importa com nada disso, você pode remover absolutamente um monte de arquivos.

rm -f file1

A opção -f ou force diz ao rm para remover todos os arquivos, estejam eles protegidos contra gravação ou não, sem avisar o usuário (contanto que você tenha as permissões apropriadas).

rm -i arquivo

Adicionar o sinalizador -i como muitos dos outros comandos, lhe dará um aviso se você deseja realmente remover os arquivos ou diretórios.

rm -r pasta

aviso: pasta e diretorio e a mesta coisa.. Você não pode simplesmente da um rm em um diretório por padrão, você precisará adicionar o sinalizador -r (recursivo) para remover todos os arquivos e quaisquer subdiretórios que possa ter. Você pode remover um diretório com o comando rmdir.

rmdir diretorio

find

Com todos esses arquivos que temos no sistema, pode ser um pouco agitado tentar encontrar um arquivo específico. Bem, há um comando que podemos usar para isso, encontre!

find /home -name puppies.jpg

Com find você terá que especificar o diretório que irá pesquisar, o que você está procurando, neste caso estamos tentando encontrar um arquivo com o nome de puppies.jpg. Você pode especificar o tipo de arquivo que está tentando encontrar.

find / home -type d -name minha_pasta

Você pode ver que eu defini o tipo de arquivo que estou tentando encontrar como (d) para diretório e ainda estou pesquisando pelo nome de minha_pasta. Uma coisa legal a se notar é que o find não para no diretório que você está procurando, ele vai olhar dentro de qualquer subdiretório que o diretório possa ter.

help

O Linux tem algumas ótimas ferramentas integradas para ajudá-lo a usar um comando ou verificar quais sinalizadores estão disponíveis para um comando. Uma ferramenta, help, é um comando bash embutido que fornece ajuda para outros comandos bash (echo, logout, pwd, etc).

help echo

Isso lhe dará uma descrição e as opções que você pode usar quando quiser executar o echo. Para outros programas executáveis, é convenção ter uma opção chamada --help ou algo semelhante.

echo --help

Nem todos os desenvolvedores que distribuem executáveis estão em conformidade com esse padrão, mas provavelmente é sua melhor chance de encontrar ajuda para um programa.

man

Puxa, gostaria que alguns desses programas tivessem um manual para que possamos ver mais algumas informações sobre eles. Bem, felizmente eles fazem! Páginas man apropriadamente nomeadas, você pode ver os manuais para um comando com o comando man.

man ls

As páginas de manual são manuais integrados por padrão na maioria dos sistemas operacionais Linux. Eles fornecem documentação sobre comandos e outros aspectos do sistema. Experimente em alguns comandos para obter mais informações sobre eles.

alias

Às vezes, digitar comandos pode se tornar realmente repetitivo, ou se você precisar digitar um comando longo muitas vezes, é melhor ter um apelido que você possa usar para isso. Para criar um alias para um comando, você simplesmente especifica um nome de alias e define-o para o comando.

alias foobar = 'ls -la'

Agora, em vez de digitar ls -la, você pode digitar foobar e ele executará esse comando, uma coisa bem legal. Lembre-se de que este comando não salvará seu alias após a reinicialização, portanto, você precisará adicionar um alias permanente em:

~ / .bashrc

ou arquivos semelhantes se quiser que persistam após a reinicialização. Você pode remover aliases com o comando unalias:

unalias foobar
